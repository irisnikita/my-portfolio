# i18n guide

## Goal
One i18n system, no duplicated logic.

## Where strings live
- `src/i18n/locales/en/*.ts`
- `src/i18n/locales/vi/*.ts`

Split by feature/topic (examples):
- `nav.ts` – navigation labels
- `home.ts` – homepage strings
- `common.ts` – shared UI strings (a11y, chatbot, etc.)

`src/i18n/messages.ts` merges them into a single dictionary per language.

## How to use in Astro
```ts
import { getLangFromUrl, useTranslations, useTranslatedPath } from "../i18n/utils";

const lang = getLangFromUrl(Astro.url);
const t = useTranslations(lang);
const translatePath = useTranslatedPath(lang);
```

## Routing model
Routes are defined once via `src/pages/[[lang]]/...`.

Static build needs every locale path generated:
- For non-dynamic pages: `export { getStaticPaths } from "../../i18n/routing";`
- For slug pages: generate `{ slug }` × `{ locale }` combinations.

## Rules
- No component-level language detection (`pathname.startsWith('/vi')`).
- No inline bilingual objects.
- Add new keys in EN first, then implement VI.
